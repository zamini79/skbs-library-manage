## 1. 프로젝트 개요

### 1.1 배경
회사 1층에 비치된 도서(약 701권)를 구성원들에게 대여하는 사내 도서 대출 서비스를 운영하고자 함. 수기 관리의 한계를 극복하고, 도서 재고/대여 현황을 체계적으로 관리하며, 구성원이 자유롭게 도서를 조회·이용할 수 있는 환경을 구축한다.

### 1.2 목적
- 도서 재고 및 대여 현황의 실시간 관리
- 관리자의 운영 효율 향상 (대여/반납/연체 처리 자동화)
- 구성원의 도서 접근성 향상 (언제든 조회 가능)
- 마일리지 제도를 통한 정상 반납 문화 정착

### 1.3 범위
**포함**: 도서 마스터/재고 관리, 대여·반납 처리, 조회·통계, 대시보드, 마일리지 적립/랭킹, 구성원 회원가입·로그인, 도서 조회 페이지, 엑셀 일괄 업로드, ISBN 자동 조회

**제외 (이번 단계)**: 연체 알림 메일 자동 발송, 마일리지 사용처/경품 교환 기능

## 2. 사용자 및 권한

| 구분 | 계정 ID | 권한 | 주요 활동 |
|---|---|---|---|
| 마스터 관리자 | `cs_admin` | 도서 마스터 관리 + 대여 관리 + 시스템 설정 | 도서 등록/수정/폐기, 엑셀 일괄 업로드, 사용자 관리, 통계 확인 |
| 대여 관리자 | `book_admin` | 대여/반납 처리 | 1층에서 책 전달 시 대여 등록, 반납 처리 |
| 구성원 | 회사 이메일(@sk.com) | 조회 | 도서 검색·조회, 본인 대여현황 확인, 마일리지 랭킹 확인 |
| 비로그인 사용자 | - | 보유 도서 목록 조회만 | 회사 도서 보유 현황 열람 |

> **참고**: 도서 마스터 변경(등록/삭제/단가 수정 등)은 마스터 관리자만 가능. 대여 관리자는 운영 활동만 수행.

## 4. 기능 명세

### 4.1 관리자 시스템

#### 4.1.1 로그인
- ID(`cs_admin` / `book_admin`) + 비밀번호 인증
- 비밀번호 변경 기능
- 권한별 메뉴 노출 분기

#### 4.1.2 대시보드 (메인 화면)
시스템 접속 시 기본 표시 화면.

**상단 KPI 카드 (4개)**
- 전체 도서 수 (보유 권수)
- 대여 중인 도서 수
- 연체 도서 수 (강조 표시, 빨간색)
- 이번 달 신규 대여 건수

**중단 차트 영역**
- 개인별 대여현황 TOP 10 (가로 막대 그래프)
- 도서별 대여현황 TOP 10 (가로 막대 그래프)

**하단**
- 최근 대여 내역 (최근 10건)
- 연체 도서 목록 (Quick View, 클릭 시 상세 페이지 이동)

#### 4.1.3 도서 재고 관리 (마스터 관리자 전용)

**도서 목록 화면**
- 전체 도서 리스트 (페이지네이션)
- 검색 필터: 제목, 저자, 출판사, 단가 범위, 수량, 상태(가용/대여중/폐기), 카테고리
- 정렬: 제목, 등록일, 대여 횟수
- 각 도서별 현재 대여 상태 표시 (대여자/대여일/반납기한)
- 일괄 폐기 기능 (체크박스 다중 선택)

**도서 신규 입고**
- **수동 입력** (기본 방식): 제목, 저자, 출판사, ISBN, 단가, 수량, 카테고리, 입고일, 표지이미지(수동 업로드)
- **ISBN 자동 조회 보조 기능**:
  - 등록 폼에서 ISBN 입력 후 `조회` 버튼 클릭
  - 외부 API(알라딘 OpenAPI 등)에서 제목/저자/출판사/표지 자동 채움
  - 채움 후에도 관리자가 수동 수정 가능
  - 표지는 자동 조회된 URL을 기본값으로 사용하되, 수동 업로드로 교체 가능 (수동 업로드가 우선)
- **엑셀 일괄 업로드**:
  - 템플릿 다운로드 제공 (제목, 저자, 출판사, ISBN, 단가, 수량, 카테고리 컬럼)
  - 파일 업로드 → 미리보기 → 검증(오류 행 표시) → 확정 등록
  - 카테고리 값 검증 (사전 정의된 5종 외에는 오류 처리)
  - 중복 ISBN 처리 정책: 기존 도서 수량 증가 vs 별도 등록 선택

**도서 폐기**
- 폐기 사유 입력 (분실/파손/노후/기타)
- 폐기 처리 후에도 과거 대여 이력 조회는 유지
- 대여 중인 도서는 반납 후 폐기만 가능

**도서 상세 페이지**
- 도서 정보 + 대여 이력 전체 (대여자, 대여일, 반납일, 연체 여부)
- 누적 대여 횟수

#### 4.1.4 도서 대여 관리

**대여 등록 (대여 관리자 + 마스터 관리자)**
- 입력 항목: 도서 검색/선택, 대여자(사번 검색), 대여일시(현재 시각 자동), 반납기한(자동 계산: +14일)
- 대여 담당자: 로그인 계정 자동 기록
- 대여 가능 여부 자동 검증:
  - 해당 도서가 가용 상태인가
  - 대여자의 당월 대여 횟수가 2회 미만인가
  - 대여자의 현재 보유 권수가 2권 미만인가
  - 대여자에게 연체 중인 도서가 있는가 (있으면 대여 불가)
- 검증 실패 시 사유 명시
- **대여 가능 시간대(평일 09:00~17:00)**: 시스템 검증 없이 운영 안내만 (담당자 부재 시 자연스럽게 대여 불가)
- **동일 도서 재대여**: 반납 직후라도 다른 제약(월 횟수, 동시 보유, 연체)을 통과하면 가능 *(추후 정책 변경 가능)*

**반납 처리**
- 도서 검색 또는 대여자 검색으로 진입
- 반납일시 자동 기록 (현재 시각)
- 연체 여부 자동 판단 → 마일리지 자동 반영
  - 정상 반납: +10점
  - 연체 반납: -5점

**대여 현황 화면**
- 전체 대여 건 목록
- 필터: 대여 중/반납 완료/연체
- 표시 항목: 도서명, 대여자(사번/부서/이름), 대여일시(YYYY-MM-DD HH:mm), 반납기한, 실제 반납일, 연체 일수, 대여 담당자
- 연체 건은 빨간색 강조, 연체 일수 표시

#### 4.1.5 조회 기능

**월별 대여현황 조회**
- 월 선택 → 해당 월 대여 건수, 반납 건수, 연체 건수
- 일별 추이 차트
- 카테고리별 대여 분포

**개인별 대여현황 조회**
- 사번/이름/부서 검색
- 해당 구성원의 누적 대여 이력, 현재 보유 도서, 마일리지 점수, 연체 이력

**도서별 대여현황 조회**
- 도서 검색
- 해당 도서의 전체 대여 이력, 누적 대여 횟수, 평균 대여 기간

#### 4.1.6 사용자 관리 (마스터 관리자 전용)
- 가입한 구성원 목록 조회
- 사번/부서/이름/이메일 검색
- 계정 비활성화 (퇴사자 처리)
- 관리자 계정 비밀번호 재설정

### 4.2 구성원 페이지

#### 4.2.1 회원가입
1. 회사 이메일(`@sk.com`) 입력 *(다른 도메인은 즉시 차단)*
2. 도메인 검증 → 인증 메일 발송 (Supabase Auth)
3. 메일 링크 클릭 → 추가 정보 입력 (비밀번호, 사번, 이름, 부서)
4. 가입 완료

> **참고**: 가입 인증 메일은 Supabase Auth 기본 기능을 사용 (별도 메일 서버 불필요). 이는 "연체 알림 메일 발송 제외" 정책과 별개임.

#### 4.2.2 로그인
- 회사 이메일 + 비밀번호
- 비밀번호 찾기 (이메일로 재설정 링크)
- 자동 로그인 옵션

#### 4.2.3 도서 조회 (메인) - **비로그인 접근 가능**
- 보유 도서 전체 리스트 (반응형 카드/리스트 뷰)
- 검색: 제목, 저자, 출판사, 카테고리
- 카테고리 필터 (5종)
- 정렬: 신규 입고순, 인기순, 제목순
- 각 도서 카드에 표시:
  - 표지, 제목, 저자, 출판사, 카테고리
  - 대여 상태 (가용 / 대여 중 + 반납 예정일)
- 비로그인 사용자에게는 상단에 "로그인하면 대여 이력과 마일리지를 확인할 수 있습니다" 안내 배너 표시

#### 4.2.4 도서 상세 - **비로그인 접근 가능**
- 도서 정보
- 현재 대여 상태
- 대여 가능 시점 (대여 중이면 예상 반납일)

#### 4.2.5 내 대여현황 - **로그인 필수**
- 현재 대여 중인 도서 (반납기한, 연체 시 D+N일 강조 표시)
- 과거 대여 이력
- 내 마일리지 점수
- 이번 달 대여 가능 잔여 횟수 표시

#### 4.2.6 마일리지 랭킹 - **비로그인 접근 가능**
- TOP 10 구성원 (이름/부서 + 점수)
- 본인의 현재 순위 표시 (로그인 시)

## 5. 정책 정의

### 5.1 대여 정책
| 항목 | 정책 |
|---|---|
| 기본 대여 기간 | 14일 (대여일 +14일 23:59:59까지) |
| 1인당 월 대여 횟수 | 최대 2회 (매월 1일~말일 기준 리셋) |
| 1인당 동시 보유 권수 | 최대 2권 |
| 대여 연장 | 불가 |
| 연체 중 신규 대여 | 불가 |
| 폐기 도서 대여 | 불가 (이력 조회만 가능) |
| **대여 가능 시간대** | **평일 09:00 ~ 17:00 (KST) — 운영 안내, 시스템 검증 없음** |
| **동일 도서 재대여** | **가능 (다른 제약 통과 시)** *추후 변경 가능성* |

### 5.2 마일리지 정책
| 상황 | 점수 |
|---|---|
| 정상 반납 | +10점 |
| 연체 반납 (연체 일수 무관) | -5점 |
| 누적 방식 | 합산, 음수 허용 |
| 적립 시점 | 반납 처리 시 자동 |

### 5.3 연체 처리 정책
- 매일 자정(00:00 KST) Vercel Cron이 실행되어 반납기한 경과 대여 건을 `연체` 상태로 자동 전환
- 연체 일수는 화면 표시 시 `현재시각 - 반납기한`으로 실시간 계산
- 메일 알림은 발송하지 않음 (시각적 표시 및 관리자 오프라인 안내로 대응)

### 5.4 도서 카테고리
다음 5종으로 고정:
1. **철학/종교/인문**
2. **사회과학**
3. **음반**
4. **문학**
5. **역사/여행**

> DB에서는 enum 또는 별도 카테고리 마스터 테이블로 관리. 엑셀 업로드 시 이 5종 외의 값은 오류 처리.

### 5.5 관리자 비밀번호 정책
| 항목 | 정책 |
|---|---|
| 최소 길이 | 6자 이상 |
| 필수 포함 문자 | 숫자 1개 이상 + 특수문자 1개 이상 |
| 만료 기간 | 없음 |
| 저장 방식 | bcrypt 해시 |

### 5.6 표지 이미지 처리 정책
- **기본 방식**: 관리자가 수동으로 이미지 파일 업로드 (Supabase Storage 저장)
- **보조 기능**: ISBN 입력 시 외부 API에서 표지 URL 자동 조회 후 미리보기 제공
- **우선순위**: 수동 업로드 이미지 > 자동 조회 URL
- 이미지가 없는 도서는 기본 플레이스홀더 이미지 표시

## 6. 데이터베이스 설계 (Supabase / PostgreSQL)

### 6.1 주요 테이블

**`users`** — 구성원
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | uuid (PK) | Supabase Auth user_id 연결 |
| email | text | 회사 이메일 (@sk.com) |
| employee_no | text | 사번 |
| name | text | 이름 |
| department | text | 부서 |
| mileage | int | 누적 마일리지 |
| is_active | bool | 활성 여부 |
| created_at | timestamptz | 가입일시 |

**`admins`** — 관리자
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | uuid (PK) | |
| login_id | text | `cs_admin` / `book_admin` |
| password_hash | text | bcrypt |
| role | text | `master` / `book` |
| name | text | |

**`books`** — 도서 마스터
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | uuid (PK) | |
| title | text | 제목 |
| author | text | 저자 |
| publisher | text | 출판사 |
| isbn | text | ISBN (nullable) |
| price | int | 단가 |
| category | text | 5종 enum |
| cover_url | text | 수동 업로드된 표지 URL |

### 6.2 권한 (Row Level Security)
- 구성원은 본인 데이터(rentals, mileage)만 조회 가능
- 도서 목록은 anonymous 사용자에게도 SELECT 허용 (비로그인 조회 지원)
- 관리자는 별도 인증 경로(서버 사이드)로 전체 데이터 접근

## 10. 향후 확장 (이번 범위 외)

### 10.1 마일리지 사용처
현재는 적립/랭킹만 표시하지만, 향후 다음과 같은 활용을 고려할 수 있음.

**아이디어 예시**
- 분기별 TOP 랭커 시상 (사내 카페 쿠폰, 도서 구매권 등)
- 마일리지로 신규 도서 신청 우선권 부여
- 누적 마일리지로 사내 굿즈/경품 교환
- 부서별 마일리지 합산 랭킹 → 부서 대항 이벤트

**필요 기능**
- 경품 카탈로그 관리
- 마일리지 차감/교환 처리
- 교환 신청 승인 워크플로우

### 10.2 알림 기능 (향후)
- 연체 알림 메일 (이번 단계 제외, 추후 도입 가능)
- 사내 메신저(Teams) 연동 — A.Biz와의 연계 가능성
- 반납기한 임박 사전 안내
- 신규 도서 입고 알림 (관심 카테고리 구독)

### 10.3 운영 고도화 (향후)
- 도서 추천 기능 (대여 이력 기반)
- 구성원의 도서 신청/리뷰 기능
- 동일 도서 연속 재대여 제한 정책 도입 검토
